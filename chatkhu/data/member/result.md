# members/export-task API 분석 결과

---

## 핵심 컬럼 의미

| 컬럼 | 의미 | 주의 |
|------|------|------|
| `소속 기간` | API 호출 시 지정한 `start_date` 값 — 멤버 속성이 아닌 **조회 날짜 메타 정보** | 8,181명 전원 동일값(`2026-06-15`) 확인. 최초 가입일과 무관 |
| `기간 내 크레딧 사용량 총합` | `start_date`~`end_date` 기간 동안 사용한 크레딧 합계 (조직+개인 모두 포함) | credit_source 파라미터로 분리 불가 |
| `기간 내 메시지 수` | 동일 기간 동안 전송한 메시지 수 | |
| `현재 월 크레딧 할당량` | **다운로드 시점** 기준 이번 달 할당량 | 파일 날짜 기준 아님 — 역사 데이터로 사용 불가 |
| `현재 월 크레딧 잔여량` | **다운로드 시점** 기준 이번 달 잔여량 | 동일 유저가 5/1·5/31 파일에서 동일값 → 역사 데이터 아님 확인 |
| `개인 구매 크레딧` | 현재 개인 구매 크레딧 잔액 | 다운로드 시점 기준 |
| `개인 구매 크레딧 사용량` | **평생 누적** 개인 구매 크레딧 사용량 | 기간 파라미터 무관 — 일별 파일 합산 시 N배 오류 |
| `학번/사번` | 학번 또는 사번 | |

## 날짜 파라미터 동작

`start_date` / `end_date`가 두 가지 역할을 동시에 함:

| 역할 | 기준 | 설명 |
|------|------|------|
| 멤버 목록 범위 | `end_date` | 최초 가입일 ≤ end_date인 **누적 전체 멤버** 표시 |
| 사용량 집계 기간 | `start_date` ~ `end_date` | `기간 내 크레딧 사용량 총합` = 이 기간의 실제 사용량 |

> `start_date=end_date=2026-05-09`이면 → 5월 9일까지 가입한 전체 멤버(6,308명)의 **5월 9일 하루** 사용량

날짜별 누적 멤버 수 (검증):

| end_date | 멤버 수 |
|----------|--------|
| 2025-11-03 | 1 |
| 2025-12-01 | 84 |
| 2026-01-02 | 236 |
| 2026-03-01 | 593 |
| 2026-05-09 | 6,308 |
| 2026-06-09 | 7,840 |

## credit_source 파라미터 비교 (2026-06-15 기준)

### dashboards/usage/export-task — 파라미터 **정상 동작** ✅

| credit_source | 합계 |
|--------------|------|
| 없음 (전체) | 590,391.263 |
| organization | 397,449.384 |
| personal | 192,941.879 |
| org + personal | **590,391.263** (전체와 일치 ✅) |

### members/export-task — 파라미터 **완전 무시** ❌

| credit_source | 멤버 수 | 합계 | 학생 | 교수 | 교직원 |
|--------------|--------|------|------|------|------|
| 없음 (전체) | 8,181 | 590,391.264 | 420,633.812 | 64,108.623 | 105,648.829 |
| organization | 8,181 | 590,391.264 | 420,633.812 | 64,108.623 | 105,648.829 |
| personal | 8,181 | 590,391.264 | 420,633.812 | 64,108.623 | 105,648.829 |

세 variant 완전 동일 → 유저 목록도, 사용량 값도 파라미터 무관.

> members API의 전체 합계(590,391.264)는 dashboard API 전체(590,391.263)와 소수점 셋째 자리까지 일치 → `기간 내 크레딧 사용량 총합` = 조직+개인 전체 사용량이 맞음.

## 결론

| 목표 | 가능 여부 | 방법 |
|------|---------|------|
| 월별 전체 크레딧 사용량 | ✅ | members API 일별 파일 합산 또는 dashboard API |
| 월별 조직 크레딧만 | ✅ | dashboard API (`credit_source=organization`) |
| 월별 개인 크레딧만 | ✅ | dashboard API (`credit_source=personal`) |
| 신분별 크레딧 사용량 | ✅ | members API `신분` 컬럼 groupby (단, 조직+개인 혼합) |
| **신분별 × 조직 크레딧** | ❌ | 현재 불가 — members API에서 credit_source 무시됨 |

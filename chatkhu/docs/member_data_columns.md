## API 1: dashboards/usage/export-task

### 엔드포인트

```
GET /api/proxy/admin/dashboards/usage/export-task/
```

### 파라미터

| 파라미터 | 필수 | 값 예시 | 설명 |
|---------|------|--------|------|
| `response_type` | ✅ | `csv` / `xlsx` | 응답 파일 형식 |
| `start_date` | ✅ | `2026-05-01` | 집계 시작일 (YYYY-MM-DD) |
| `end_date` | ✅ | `2026-05-31` | 집계 종료일 (YYYY-MM-DD) |
| `credit_source` | ✅ | `organization` / `personal` | 크레딧 출처 필터 — **이 API에서는 실제로 적용됨** |

> `credit_source=organization` → 조직 크레딧 풀에서 차감된 금액만 집계 (정확한 org-only 값)

### 응답: 비동기 task 방식

```
POST → task_id 반환
GET /api/proxy/tasks/{task_id}/  # status: pending → completed
GET /api/download-media?url=...  # 완료 후 파일 다운로드
```

### 응답 CSV 구조

CSV는 **멀티섹션** 포맷. 섹션마다 헤더행 + 데이터행 반복.

#### 섹션 1: 전체 합계

```

397449.384,305582.804,977.0,12880.674,8730.0,349.562,1507.532,0.0,0.0,4605.149,0.0,1830.202,60986.462
```

| 컬럼 | 의미 |
|------|------|
| `합계` | **기간 내 전체 크레딧 사용량** (조직 크레딧만, `credit_source=organization` 기준) |
| `채팅` | 채팅(LLM) 모델 사용 크레딧 합계 |

| `오디오` | 오디오 관련 크레딧 |
| `딥 리서치` | 딥 리서치 기능 크레딧 |
| `에이전트` | Super Agent 등 에이전트 크레딧 |

> **크로스체크 코드에서 사용하는 값**: 헤더행 `합계,...` 다음 줄의 첫 번째 컬럼 (`crosscheck_monthly_credits.py` 참조)

```
모델,입력 크레딧,출력 크레딧,웹 검색 크레딧,입력 토큰,출력 토큰,웹 검색 횟수

...
```

| 컬럼 | 의미 |
|------|------|
| `모델` | LLM 모델명 |
| `입력 크레딧` | 입력 토큰 소비 크레딧 |
| `출력 크레딧` | 출력 토큰 소비 크레딧 |
| `웹 검색 크레딧` | 웹검색 소비 크레딧 |
| `입력 토큰` | 입력 토큰 수 |
| `출력 토큰` | 출력 토큰 수 |
| `웹 검색 횟수` | 웹검색 요청 횟수 |

#### 섹션 3: 이미지 모델별 상세

```
이미지 모델,생성 횟수,사용된 크레딧
Nano Banana Pro,8,1134.75
...
```

#### 섹션 4: 비디오 모델별 상세

```
비디오 모델,생성 횟수,사용된 크레딧
Sora 2,1,405.0
...
```

#### 섹션 5–N: 기능별 상세

각 특수 기능(Korea-in-Data, Univ-in-Data, PPT 생성, 딥 리서치, 에이전트)별로:
```
{기능명},사용된 크레딧
{기능명},{값}
```

### 한계

- **신분별(학생/교수/교직원) 분해 불가** — 전체 합계만 제공
- 유저별 명세 없음

## API 2: members/export-task

### 엔드포인트

```
GET /api/proxy/admin/members/export-task/
```

### 파라미터

| 파라미터 | 필수 | 값 예시 | 설명 |
|---------|------|--------|------|
| `response_type` | ✅ | `csv` / `xlsx` | 응답 파일 형식 |
| `start_date` | ✅ | `2026-05-09` | 사용량 집계 시작일 (YYYY-MM-DD) |
| `end_date` | ✅ | `2026-05-09` | 사용량 집계 종료일 + 멤버 목록 기준일 (YYYY-MM-DD) |
| `credit_source` | ❌ 무효 | `organization` | **유저 필터도 사용량 필터도 아님 — 이 API에서는 완전히 무시됨** (검증 완료 — 아래 주의사항 참조) |

### 날짜 파라미터 동작 방식 ⚠️

`start_date`와 `end_date`가 두 가지 역할을 동시에 함:

| 역할 | 기준 파라미터 | 설명 |
|------|------------|------|
| **멤버 목록 범위** | `end_date` | `최초 가입일 ≤ end_date`인 전체 누적 멤버를 표시 |
| **사용량 집계 기간** | `start_date` ~ `end_date` | `기간 내 크레딧 사용량 총합` = 이 기간 동안의 실제 사용량 |

```
예시: start_date=2026-05-09, end_date=2026-05-09 (하루)
  → 멤버 목록: 2026-05-09까지 가입한 전체 누적 멤버 (6,308명)
  → 기간 내 크레딧 사용량 총합: 각 멤버가 5월 9일 하루에 쓴 크레딧
```

실제 검증 결과 (날짜별 멤버 수 추이):

| end_date | 멤버 수 | 최초 가입일 범위 |
|----------|--------|----------------|
| 2025-11-03 | 1 | 2025-05-13 ~ 2025-05-13 |
| 2025-12-01 | 84 | 2025-05-13 ~ 2025-12-01 |
| 2026-01-02 | 236 | 2025-05-13 ~ 2026-01-02 |
| 2026-03-01 | 593 | 2025-05-13 ~ 2026-03-01 |
| 2026-05-09 | 6,308 | 2025-05-13 ~ 2026-05-09 |
| 2026-06-09 | 7,840 | 2025-05-13 ~ 2026-06-09 |

> `download_daily_org.py`가 하루씩 `start_date=end_date=그날`로 파일을 수집하는 이유:  
> 하루 단위 사용량 × 31일 합산 = 월별 총합. 단, 새로 가입한 멤버는 가입 후 파일부터만 집계됨.

> ⚠️ **`credit_source` 파라미터 주의**: 2026-05-09 기준 파라미터 있음 vs 없음 비교 결과:
> - 유저 집합: 6,308명 vs 6,308명 (추가/제외 0명 — 유저 필터 효과 없음)
> - 개인 구매 크레딧 보유 유저 개별 총합값: 완전 동일 (사용량 필터 효과 없음)
> - 신분별·전체 합계: 157,550.424 vs 157,550.424
>
> **→ 유저 목록도, 각 유저의 사용량 값도 파라미터 유무와 무관하게 동일. 이 API에서 파라미터 완전 무효.**  
> 브라우저 UI도 멤버 CSV 다운로드 시 이 파라미터를 전송하지 않음 (네트워크 로그 확인).

## credit_source 파라미터 동작 차이

`credit_source=organization` 파라미터는 **엔드포인트마다 동작이 다름**.

| 엔드포인트 | credit_source 효과 | 비고 |
|-----------|-------------------|------|
| `dashboards/usage/` (화면 카드) | ✅ 정상 동작 — 조직 크레딧만 집계 | 브라우저 탭 전환 시 숫자 변화 |
| `dashboards/usage/export-task/` (개요 CSV) | ✅ 정상 동작 | 위와 동일 엔드포인트 |
| `members/export-task/` (멤버 CSV) | ❌ 완전 무시 | UI도 이 파라미터 안 보냄 (네트워크 로그 확인) |

> 브라우저에서 "조직 제공" 탭 클릭 시 화면 숫자가 달라지는 건 `dashboards/usage/?credit_source=organization`을 호출하기 때문.  
> 같은 상태에서 멤버 CSV 다운로드를 눌러도 `members/export-task`는 credit_source 없이 호출됨.

## 두 API 비교 요약

| 항목 | dashboards/usage/export-task | members/export-task |
|------|------------------------------|---------------------|
| 조직 크레딧만 필터 | ✅ `credit_source=organization` 유효 | ❌ 파라미터 무시됨 |
| 신분별 분해 | ❌ 불가 | ✅ `신분` 컬럼으로 가능 |
| 유저별 명세 | ❌ | ✅ |
| 모델별 상세 | ✅ (집계) | ✅ (유저별) |
| 응답 구조 | 멀티섹션 CSV | 행=유저 단일 테이블 |

**→ 신분별 × 조직 크레딧 월별 합계는 두 API 어느 쪽으로도 현재 불가.**
